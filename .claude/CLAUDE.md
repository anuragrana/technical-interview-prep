# Maintainer notes — architect-reading-list.md

## Why this file exists
`architect-reading-list.md` is a curated reading list for a practicing software architect.
It grew out of a real question: how do you discover the paradigms, processes, and tools other
tech companies use, so you can bring the good ones into your own org? Rather than let that get
buried in chat, it became a durable, curated list.

It is deliberately **broader than interview prep** (the rest of this repo). This list is day-job
reading: how real orgs build, ship, and organize — Architecture & System Design, Dev Process &
Delivery, Org & Culture, plus blogs/sources worth following.

## Curation bar (apply when adding more)
- **Free** — no paywall. (Explicitly avoiding Medium-style paywalled links; this is the free alternative.)
- **Evergreen** — holds up over years, not news.
- **Genuinely useful & interesting** to an architect, not filler.
- When asked to "pick something," lead with **one** strong recommendation, then a short shelf.
- Tie reads to a real problem; don't cargo-cult big-company practices.

## House style (match the existing repo)
- Terse `###` / `####` section headers, `- [Title](url)` bullets.
- Short parenthetical notes like `(best)` — no tables, no checkboxes.
- See `README.md` and `system-design-topics.md` for the existing style.
- Link to the user's other repos (e.g. `Engineering-Tech-Blogs`) instead of duplicating lists.
